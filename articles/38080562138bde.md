## きっかけ

個人開発でGitHub APIからStar数やContributor数を取得する処理を書いていて、何の気なしに `GITHUB_TOKEN` を環境変数にセットしようとしたところで、ふと手が止まった。

自分が普段使っているGitHubアカウントは、会社のOrganizationにも所属している。
このアカウントでPersonal Access Token（PAT）を発行して、それをローカルの個人開発環境に置く——以前までは何も考えずにやっていたが、会社のアカウントとして使用始めて以降、より慎重にこの辺意識するようになった。

万が一でも会社の資産漏えいとかにつながってしまったらとんでもないことであるため、しっかり調べないとと思ったのがきっかけ。


## PATは二種類

GitHubの設定画面でPATを発行しようとすると、Tokens (classic) と Fine-grained tokens の2種類が選べる。

Classic PATの問題は、リポジトリを絞り込めないことにある。`repo`スコープにチェックを入れると、それは「自分のアカウントがアクセスできる全リポジトリ」への読み書き権限になる。つまり会社のOrganizationの非公開リポジトリも例外なく含まれる。
今回読みたいのはNext.jsやRemixの公開リポジトリの統計情報だけなのに、この設定だと会社のコードベースまで触れる鍵を一つのトークンに同居させることになる。漏えいしたときの被害範囲がまるで違う。

ここでFine-grained PATに切り替えれば安全、という結論に最初は飛びつきかけたが、もう一段階引っかかるところがあった。

## 「自分のリポジトリを選ぶ」では解決しない

Fine-grained PATのRepository accessには「All repositories」と「Only select repositories」という選択肢がある。素直に「Only select repositories」を選んで、対象のリポジトリだけ指定すればいいと思っていた。

ところが実際に設定画面を開いてみると、選択候補のリストに `vercel/next.js` や `remix-run/remix` が出てこない。当然で、このリストは「自分が所有・管理しているリポジトリ」からしか選べない。今回読みたいのは自分のリポジトリではなく、完全に無関係な第三者の公開リポジトリだ。「特定のリポジトリに絞る」という発想そのものが、この用途には噛み合っていなかった。

正解は、Repository accessにもう1つ用意されている **「Public Repositories (read-only)」** という選択肢だった。これは個別のリポジトリを選ぶものではなく、「GitHub上の全公開リポジトリへの読み取りのみ」を一括で許可する、独立したカテゴリ。会社のOrganizationの非公開リポジトリは公開されていない以上、この権限では原理的に触れない。個人で所有しているリポジトリへの読み書きとは別物として用意されている、という点に気づいてようやく腑に落ちた。

## もう一段の安全策：SSO認可ダイアログ

会社のOrganizationがSAML SSOを使っている場合、トークンをそのOrgに対して明示的に「authorize（認可）」しないと、そのOrgのリソースにはアクセスできない仕組みもある。Public Repositories (read-only)で発行している以上ここで困ることはないはずだが、トークン発行時にOrgへの認可を求めるダイアログが出ても認可しない、というのを意識しておくと二重の安全策になる。

## ローカルでの扱いは結局シンプルな話

トークンの権限さえ正しく絞れていれば、ローカルでの扱いは通例通りで十分。

- `.env` に書いて、`.gitignore` に `.env` を入れてコミット対象から外す
- コード中に直接文字列で書かない

これは個人開発を始める時点で `.gitignore` に `.env` を入れておけば自然と守れる。
今回の本質は「ローカルでの扱い方」より「発行するトークンの権限設計」の方にあった。

## まとめ

会社のOrganizationに所属しているアカウントで個人開発をすること自体は、特別問題はない。
ただトークンの扱いは注意したい。

- 自分のリポジトリを読み書きしたい → Only select repositoriesで対象を指定
- 他人の公開リポジトリの情報を読みたいだけ → Public Repositories (read-only)

地味な気づきだが、同じようにOrganization所属のアカウントで個人開発をしている人には役立つはずだと思い、メモを残しておく。
